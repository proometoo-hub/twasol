Twasol Pro v6.45.0 - Security Hardening + Architecture Cleanup

تم تنفيذ هذه المرحلة مباشرة فوق آخر نسخة v6.44.5 وتشمل:

1) تخزين Hash للجلسات داخل قاعدة البيانات بدل حفظ التوكن الخام.
2) منع API الجلسات من إرجاع التوكنات، وإرجاع isCurrent فقط.
3) تشفير PIN باستخدام bcrypt وعدم إرجاعه للواجهة، مع hasLockedPin و verify-pin.
4) حماية webhooks للإداريين فقط مع إخفاء السر وعدم إرجاعه صريحًا.
5) حماية seed الخاص بالملصقات ليصبح Admin only.
6) تحويل محفظة مالية داخل transaction لتقليل race conditions.
7) نقل Push Tokens من ملف JSON إلى جدول SQLite داخلي باستخدام raw SQL مع ترحيل تلقائي.
8) تشديد التحقق في signaling للمكالمات الفردية والجماعية بحيث لا يتم إرسال offer/answer/ice إلا لأعضاء نفس المحادثة.
9) تحسين رفع الملفات عبر فحص magic bytes وحذف الملف عند عدم تطابق المحتوى.
10) تشديد static headers للملفات المرفوعة.
11) تقليل كشف تفاصيل الأخطاء في production.
12) فصل تخزين التوكن على الجوال إلى SecureStore بدل AsyncStorage.
13) نقل جلسة الويب من localStorage إلى sessionStorage.
14) تعطيل HTTP cleartext افتراضيًا في نسخة الموبايل.
15) حذف مفاتيح وشهادات التطوير الخاصة من الحزمة.
16) تعطيل first-user admin fallback افتراضيًا إلا إذا تم تفعيله صراحة.
17) إعدادات .env و frontend defaults أكثر أمانًا ومرتكزة على HTTPS.

ملاحظة مهمة:
هذه النسخة أصبحت أصلب أمنيًا بشكل واضح، لكنها ما زالت تستفيد أكثر لو تم لاحقًا:
- الانتقال إلى PostgreSQL
- إضافة اختبارات تلقائية
- تقسيم App.js الضخم على الموبايل
- اعتماد Refresh Token/Cookies آمنة للويب
